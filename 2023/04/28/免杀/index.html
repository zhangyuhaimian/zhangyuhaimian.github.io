免杀

基本免杀

  1. 杀毒软件的基本等级

    1. 无害
    2. 可疑
    3. 确认病毒

·静态
通常通过反编译的方式查看源代码

  1. 代码中存在函数 1.exe ,cshellcode ,java shellcode,virtualalloc,rtlmovememory,ntcreatthread等
    主要都是 windowsapi函数，尤其是和内存，堆，线程相关的函数，
    在python中存在 cmd等关键词也会被识别，如 subproceess。popen(“cmd/c”) 可以改为subprocess。popen(“命令”)
  2. shellcode特征
  3. 文件名称或md5
  4. 加密（可疑）
    使用加密解密行为或对文件又额外保护措施

·动态

通常是静态分析之后，部分杀软会有沙盒（也叫启发式查杀，通过模拟计算机的环境执行目标文件观察特征行为）
（内存小，时间快，减少对计算机的消耗，io设备缺失（鼠标和键盘））

https://www.virustotal.com/gui/home/upload
https://s.threatbook.com/

判断是不是 一般沙箱 10 到20 s 沙河内部速度比现实世界快，提高查杀速度

可以在马中 加一个休眠 不立即使用

·计算机相关

通常由r1 r2 层挂监控的方式 类似（hook） 触发这些条件就会产生事件

  1. 服务
  2. 注册表
  3. 组策略
  4. 防火墙
  5. 敏感程序

·用户
添加删除修改等操作文件夹

C:/windows/system32

C:\Users\Administrator\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup

C:\tmp等敏感文件夹

例 ： cmd powershell wmi psexec bitsadmin rundll

常见绕过思路、

·白名单调用这些敏感行为，再导入恶意内容

·网络相关

socket ==> 控制端 shellcode 小马建立链接
控制端 ==> 客户端 控制端给客户端马
客户端 ==> 控制端 客户端回连

小马拉大马

http /uri 请求到控制端 符合控制端算法 确实是他的马发送的交互式特征
控制端把大马放到客户端内存中 客户端上线

IP****，域名，证书匹配

查找通讯的ip或域名是否之前存在攻击行为

流量内容

时间特征：扫描等

内容特征：data字段中是否存在命令相关关键词或关键词加密特征

结构特征：是否存在已知远控的通讯结构特征

常见的绕过思路

tcp分段，内容加密，使用合法证书等

总结

不是所有的杀毒软件都有这些，有些比较拉跨，有些比较强，不过总体的思路都是差不多的。

对编程语言的免杀方式

powershell-》混淆、加密

c++-》编译过程、混淆、加密

python-》混淆、加密

通用-》启发式查杀

编译jar
jar cvfm ./bin/fileupload.jar ./bin/MANIFEST.MF -C ./bin/classes ./net
./bin/fileupload.jar 要生成文件位置
./bin/MANIFEST.MF 这个文件的路径
./bin/classes 指定那个目录又class文件
./net 起始位置

远控

反编译exe，IAT导出表。IAT导出表可以查看当前程序使用了哪些函数。loader加载器
通过DLLImport 直接导入

MemeryCopy //向创建的内存空间写入自己的shellcode
